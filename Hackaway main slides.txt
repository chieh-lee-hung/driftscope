Universal language and a set of rules that allow different systems (APIs and Agents) to be discovered and communicate
Now, let's introduce protocols. Think of a protocol as a universal language, a set of rules that everyone agrees on. It's like the difference between every country having its own unique power outlet versus having a universal standard like USB-C.
With protocols, our agent doesn't need to know the specific, low-level details of how the file system or the database API works. It just needs to know how to speak the protocol.
This gives us two massive wins:
Discoverability: The agent can ask, 'What can you do?' and get a structured, understandable answer.
Interoperability: We can swap out the backend implementation—change the database, use a different search agent—and as long as the new component speaks the same protocol, our agent code doesn't change.
In this module, we'll focus on two types of protocols: one for when our agent is talking to a tool or a data source—that's MCP. And another for when our agent is talking to another agent—that's A2A.


Google Cloud is leading the way in agent building standards 
A2A
Agent-to-Agent protocol is the open standard for collaboration between agents
ADK
Agent Development Kit is an open framework to build multi-agent solutions
AP2
Agents Payment Protocol is the open protocol to handle agentic payments
Once you move beyond models, you need to ensure that your agents can work across clouds and across organizations. This is why Google Cloud is leading the way in open agent building standards with the agent development kit, agent to agent protocol, and the agents payment protocol – all of which have hundreds of ecosystem partners already.

ADK:
Define multi-agent applications orchestrating actions across many agents and tools

Intuitive local dev UI for fast iteration; visualize agent topology and trace agent’s actions

Interleave deterministic logic with actions driven by gen AI reasoning for effortless, hybrid agents

Built-in support for long-running sync tools / human-in-the-loop

Gemini is the default, but any generative AI model is supported, including fine-tuned models 

Support to call remote agents and thousands of existing tools  via MCP

Python and Java SDKs

Roadmap is published

A2A:
Open protocol designed to handle the dynamic and long-running nature of agent collaboration
Enables agent discovery using Agent Card 

Designed with enterprise-use in mind for a secure and governed experience 
Built on top of existing, popular standards including HTTP, SSE, JSON-RPC

AP2:
AP2 is built on a few core principles to create a secure and fair ecosystem for everyone.

1. Openness and Interoperability: This is a non-proprietary, open extension for A2A and MCP. It fosters a competitive environment where anyone can innovate.
2. User Control and Privacy by Design: The user is always in control. A role-based architecture makes sure that sensitive data is only shared with the user's permission and only where it is needed. 
3. Verifiable Intent, Not Inferred Action: Transactions are anchored to deterministic, cryptographically-signed proof of intent from all parties involved, directly addressing the risk of agent error.
4. Clear Transaction Accountability: The protocol creates a non-repudiable, cryptographic audit trail for every transaction, providing the evidence needed to resolve disputes.
5. Global and Future-Proof: V0.1 supports "pull" payments like cards, but the roadmap includes "push" payments like real-time bank transfers and digital currencies, ensuring it can evolve with how the world pays. AP2 will also support payments where the human is not present, delegating authority for the payment in the future.
